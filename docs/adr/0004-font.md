# 0004 — Font

## Status

Accepted

## Context

Next.js scaffolds with Geist via `next/font/google` by default. The project needed a font that felt modern and distinctive without being overused, and that worked well across both marketing pages and dense application UI.

## Decision

Use Geist Sans and Geist Mono from the `geist` npm package, imported directly in `src/app/layout.tsx` and applied via CSS variables (`--font-sans`, `--font-geist-mono`). Geist is Vercel's purpose-built typeface, optimized for code-adjacent interfaces and highly legible at small sizes.

## Consequences

- Fonts are served from `node_modules` via the geist package rather than being auto-optimized by `next/font` — performance difference is negligible in practice
- If the font needs to change in the future, update `layout.tsx` and the CSS variable references in `globals.css`
